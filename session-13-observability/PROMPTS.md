# [Session 13] AI 활용 프롬프트 -- 관찰가능성

## 1단계: 이해

```
Prometheus의 4가지 메트릭 타입(Counter, Gauge, Histogram, Summary)을 설명해줘.
각각 어떤 상황에 사용하는지 실제 예시를 들어줘.
```

```
Observability의 3대 축(로깅, 메트릭, 분산 추적)의 차이를 설명해줘.
각각이 답하는 질문이 다르다는 점을 알려줘.
```

## 2단계: 적용

```
Spring Boot Actuator + Micrometer로 커스텀 메트릭을 노출하는 코드를 만들어줘.
요구사항:
- 주문 생성 수 (Counter)
- 주문 처리 시간 (Timer/Histogram)
- 현재 활성 주문 수 (Gauge)
그리고 이 메트릭을 Grafana 대시보드에 표시하는 PromQL 쿼리도 알려줘.
```

```
에러율 5% 초과 시 알림이 발생하는 Prometheus alert rule을 만들어줘.
for 절로 일시적 스파이크는 무시하도록 설정해줘.
```

