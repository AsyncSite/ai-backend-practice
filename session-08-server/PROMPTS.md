# [Session 08] AI 활용 프롬프트 -- 서버 기초

## 1단계: 이해

```
top 출력 결과에서 각 컬럼(PID, USER, PR, NI, VIRT, RES, SHR, %CPU, %MEM)이
뭘 뜻하는지 설명해줘. 각 값이 어떤 범위면 정상이고 어떤 범위면 문제인지도 알려줘.
```

```
OOM Killer가 뭐고 왜 발생하는지 설명해줘.
Java 앱에서 OOM Killer를 방지하는 JVM 옵션 설정을 알려줘.
```

## 2단계: 적용

```
서버 장애 원인을 체계적으로 분석하는 절차를 알려줘.
CPU -> 메모리 -> 디스크 -> 네트워크 순서로 확인할 명령어와
각 지표의 정상/경고/위험 범위를 포함해줘.
```

## 3단계: 검증

```
이 모니터링 스크립트가 놓치는 장애 시나리오가 있을까?
예를 들어 디스크 I/O 병목, 네트워크 패킷 유실, 좀비 프로세스 등
일반적인 명령어로 잡기 어려운 문제를 알려줘.
```
