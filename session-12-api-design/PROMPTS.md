# [Session 12] AI 활용 프롬프트 -- API 설계

## 1단계: 이해

```
HATEOAS가 뭔지, 실무에서 실제로 쓰는지 설명해줘.
RESTful API의 성숙도 모델(Richardson Maturity Model)도 함께 설명해줘.
```

## 2단계: 적용

```
SpringDoc으로 이커머스 상품 검색 API를 OpenAPI 3.0 문서화해줘.
필터링, 정렬, 페이지네이션 파라미터를 포함해줘.
```

```
Spring Boot에서 Bucket4j로 Rate Limiting 구현하는 코드를 보여줘.
요구사항:
- IP 기반 제한
- 분당 60회 제한
- 429 Too Many Requests 응답
```

## 3단계: 검증

```
이 API 설계에서 N+1 쿼리 문제가 발생할 수 있는 엔드포인트는?
목록 조회 시 연관 데이터를 함께 반환할 때 성능 문제를 방지하는 방법을 알려줘.
```
